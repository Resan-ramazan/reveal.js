Issue 1

Variable names are not meaningful

For example: kontrol, prime, mycounter

Variables like counter, kontrol, and mycounter could have more descriptive names.

For Example

Consider using candidate,primeSeries,limit, and primeCount respectively. This makes the code more readable and self-explanatory.

Issue 2

Unused Code

The block inside the if (prime == 0) condition is empty. If it's not needed, you can remove it to improve readability.

Array Usage

Instead of using a fixed-size array, consider using a dynamic data structure like List to store prime numbers. This way, you won't have to worry about defining a maximum size, and the code will be more flexible.

Separation of Concerns

Conside breaking down the logic into smaller functions to improve readability and maintainability.

1) IsPrime function 

					static bool IsPrime(int number)
					{
						for (int divisor = 2; divisor < number; divisor++)
						{
							if (number % divisor == 0)
							{
								return false;
							}
						}
				
						return true;
					}
2) Find Primes function

					static List FindPrimes(int limit)
						{
							List primeList = new List();
					
							for (int candidate = 2; candidate <= limit; candidate++)
							{
								if (IsPrime(candidate))
								{
									primeList.Add(candidate);
								}
							}
					
							return primeList;
						}
